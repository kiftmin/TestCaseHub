Subject: [PATCH] fix: correct test case progress tracking and execution submit
 flow

Three bugs fixed:

1. Progress showed 'Not Started' even with completed cases
   - TestCaseSelector and TesterScenarioPage computeScenarioProgress now
     treat execution.overall_result != null as definitively 'Completed'
   - Previously relied solely on step-result recomputation which could
     disagree with the submitted state

2. Case showed 'In Progress' after all steps were Pass/Fail
   - QuickWizard readyToSubmit now checks BOTH in-memory entries AND
     persisted stepResults from the server, so a reloaded page with
     all steps already saved correctly enables the Submit button
   - validateFailedSteps also checks persisted actual_result, not just
     the in-memory entry

3. Submit button never marked execution as Completed on the backend
   - Both GuidedWizard (Submit Case button) and QuickWizard (handleSubmit)
     now PATCH /executions/:id with status='completed' and overall_result
     ('passed' or 'failed') after saving step results
   - Without this, syncUseCaseStatus never saw a terminal state and the
     scenario/run progress stayed 'in_progress' indefinitely

Backend: prevent duplicate defect creation on re-submit
   - PATCH executions/:id now checks for an existing defect on the
     execution before inserting, returning the existing one if found
